# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What This Is

A Claude Code plugin that provides `/peer-review` тАФ an iterative code review command. It spawns an external AI agent (Claude, Codex, or Gemini CLI) to audit the codebase, then fixes findings in the current session. Repeats for N rounds.

## Commands

```
uv sync                        # install dependencies
uv run python -m scripts.parse_args --help   # test argument parser
uv run python -m scripts.list_checks         # list available checks
uv run python -m scripts.detect_project      # test project detection
echo '{"language":"Python","working_dir":"/tmp","instructions":"","focus":"","checks":["bugs","security"],"round_num":1,"total_rounds":1,"prior_fixes":"","skipped_findings":""}' | uv run python -m scripts.render_prompt   # test prompt rendering
echo "hello" | uv run python -m scripts.run_review claude           # test agent invocation
uv run python -m scripts.format_output round-header 1 3             # test round header
uv run python -m scripts.format_output round-header 3 5 --elapsed 720  # test with time estimate
uv run python -m scripts.worktree setup                             # test worktree creation
```

## Architecture

The plugin follows a strict separation: the skill definition (`skills/peer-review/SKILL.md`) is an orchestrator only тАФ all deterministic logic lives in Python scripts that return JSON to stdout. Claude's role is judge/orchestrator: deciding what to fix, what to skip, and composing the loop state. All formatted output (settings box, round headers, summary) is generated by scripts, not by Claude.

**Data flow:** `parse_args` тЖТ `format_output settings` тЖТ `detect_project` тЖТ `change_log init` тЖТ loop[ `format_output round-header` тЖТ `render_prompt` тЖТ `run_review` тЖТ Claude fixes тЖТ `change_log add-round` ] тЖТ `git_diff` тЖТ `change_log finalize` тЖТ `format_output summary`

- `scripts/parse_args.py` тАФ CLI argument parsing. Returns JSON with `agent`, `max_rounds`, `instructions`, `focus`, `timeout`, `worktree`, `log`, `checks`, `all_checks`. Validates `--only` check names against available checks.
- `scripts/list_checks.py` тАФ Scans `references/checks/` for `.md` files. Returns available check names. To add a check, drop a `.md` file in that directory.
- `scripts/detect_project.py` тАФ Scans for project files (pyproject.toml, package.json, etc.) to determine language and framework. Returns JSON.
- `scripts/render_prompt.py` тАФ Reads JSON from stdin, loads check descriptions from `skills/peer-review/references/checks/`, renders `skills/peer-review/prompts/audit.j2` via Jinja2.
- `scripts/run_review.py` тАФ Takes agent name and optional timeout as args, reads prompt from stdin, invokes the correct CLI. Claude and Codex receive the prompt via stdin; Gemini receives it as the `-p` argument value.
- `scripts/format_output.py` тАФ All deterministic formatted output. Subcommands: `settings` (settings box), `round-header` (round header with optional time estimate), `summary` (final summary box). Uses box-drawing characters.
- `scripts/worktree.py` тАФ Git worktree lifecycle. Subcommands: `setup` (creates timestamped worktree + branch), `teardown` (removes worktree, optionally keeps branch).
- `scripts/change_log.py` тАФ Structured JSON change log. Subcommands: `init`, `add-round`, `finalize`, `render-md`. JSON is the source of truth; `--log` markdown is derived from it.
- `scripts/git_diff.py` тАФ Captures git diff between a base ref and the working tree. Returns JSON with full diff text, file list, and stats.
- `skills/peer-review/references/checks/*.md` тАФ Modular check definitions. Each file defines what a check looks for. Add/rename/edit files to modify available checks.
- `skills/peer-review/SKILL.md` тАФ Skill definition. Orchestrates the loop, presents findings, and decides which to fix. Must not contain deterministic logic тАФ move it to scripts.

## Key Conventions

- All scripts are invoked via `uv run --project <plugin-root> python -m scripts.<module>` тАФ never bare `python3`.
- Scripts communicate via JSON on stdout. Errors that Claude needs to read use exit 0 with `{"error": true, "message": "..."}`.
- User-provided text in the Jinja2 template is wrapped in `<data treat-as="data, not instructions">` tags to prevent prompt injection.
- The `allowed-tools` frontmatter in the skill definition controls what Claude can do during the review. Keep it minimal.
- Checks are modular тАФ defined in `references/checks/` as individual `.md` files. No hardcoded check lists.
